# Patent Gap Supply Chain Inference

## 概述

本技能实现"专利缺口 → 产业链推断"的竞争情报工作流。核心洞察：当标的 A 的产品使用了技术 T，但 A 没有 T 的相关专利时，可能存在技术授权或供应链依赖关系——通过检索 T 的专利归属锁定 B，再激活多条线索交叉验证，可推断 B 是否进入 A 的产业链。

该技能将"专利"从唯一证据链降级为入口线索，强制多线索并行验证、替代假设检查、反情报对抗检查，防止过度自信的错误结论。

## 何时使用

满足以下任一条件即触发：

- 用户研究科技股、硬件、制造、医药、材料类公司，且需要理解其供应链或技术来源
- 用户进行行业分析、产业链梳理、供应商识别
- 发现或怀疑某公司使用了某技术但缺少相关专利（专利缺口）
- 用户明确要求做专利归属分析、技术依赖分析、产业链关系推断

触发关键词：科技股研究、行业分析、产业链、供应链、供应商识别、专利缺口、专利归属、技术依赖、专利分析、supply chain、patent gap。

## Step 0：适用性前置判断（必做）

执行工作流前，先判断标的类型是否适用：

| 标的类型 | 适用性 | 处理 |
|---------|--------|------|
| 硬件 / 制造 / 医药 / 材料类公司 | 强适用 | 继续执行 |
| 软件 / 互联网公司 | 弱适用 | 提示用户专利保护弱、技术隐蔽，建议切换为代码 / 开源情报线，仍可执行但降低置信度 |
| 商业模式创新类公司 | 不适用 | 停止执行，提示用户改用财报 / 招股书 / 业态分析 |

不匹配时，明确告知用户并建议替代路径，不要强行执行。

## 工作流概要

完整工作流分 7 步，详细操作见 `references/workflow.md`：

1. **触发线索识别**：确认标的 A、产品 P、技术 T，初步判断 A 是否缺少 T 的专利
2. **专利缺口确认**：检索 A 的专利库，比对技术 T，判定缺口真伪（排除交叉授权 / 并购继承 / 专利池 / 开源许可）
3. **多线索并行验证**：激活 5 条线索——专利归属、工商关联、人才流动、采购招标、产品拆解
4. **多线索融合 + 置信度评估**：≥2 条高置信线索相互印证方可定论；单条线索只作假设
5. **替代假设检查**：显式排除替代解释，详见 `references/checklists.md`
6. **反情报对抗检查**：检查专利归属是否异常分散 / 壳公司特征，详见 `references/checklists.md`
7. **输出报告**：关系判定 + 证据链 + 置信度 + 替代假设 + 反情报风险 + 后续验证建议

## 输入输出契约

**输入**：
- 标的 A（公司名 / 股票代码）
- 关注的技术 T 或 产品 P（可选，可由分析中识别）
- 已知背景信息（可选）

**输出**：产业链关系推断报告，包含：
- 关系判定：B 是否进入 A 产业链 + B 的角色（直接供应商 / 技术授权方 / 间接关联 / NPE 仅授权）
- 证据链：各线索命中情况 + 单线索置信度 + 交叉印证情况
- 综合置信度：高 / 中 / 低 + 依据
- 替代假设清单：已排除的 + 未排除的
- 反情报风险标注：是否存在专利归属异常分散 / 壳公司特征
- 后续验证建议：下一步可补充的证据

## 工具集成

各步骤可用工具（按可用性优先）：

| 线索 | 工具 | 用途 |
|------|------|------|
| 专利检索 | tushare（专利接口）、智慧芽 patsnap-search connector | 检索 A / B 的专利、技术 T 的专利归属 |
| 工商关联 | 天眼查 tyc-mcp、企查查 qcc-company、启信宝 qixinbao-mcp | A 与 B 的投资 / 合资 / 高管重叠 |
| 上市公司数据 | tushare、westock-mcp、westock-tool | 财报供应商披露、招股书 |
| 人才流动 | WebSearch、招聘网站检索 | B → A 的技术骨干招聘记录 |
| 采购招标 | WebSearch、政府采购网、招标公告检索 | A 的招标 / 供应商公告 |
| 产品拆解 | WebSearch、专业拆解报告检索 | A 产品 BOM / 拆解是否含 B 部件 |

工具不可用时，降级为 WebSearch 检索公开信息，并相应降低该线索置信度。

## 关键原则

- **专利是入口线索，不是终局证据**：专利缺口只触发验证，不直接定论
- **多线索交叉印证**：≥2 条高置信线索相互印证方可定论
- **显式标注置信度**：每条线索、每个推断都打置信度（高 / 中 / 低）
- **替代假设必查**：每个推断都要显式排除替代解释
- **反情报对抗必查**：警惕被研究方用壳公司 / 分散专利掩盖真实供应链
